# Lab Exercise Template

## Structure

Each lab exercise should follow this standardized format:

```
# [Lab Title]

## Learning Objectives

- Objective 1
- Objective 2
- Objective 3

## Prerequisites

- Software/hardware requirements
- Knowledge requirements
- Setup requirements

## Setup Instructions

1. Step-by-step setup process
2. Verification steps to confirm proper setup

## Procedure

### Step 1: [Description of what this step accomplishes]
- **Commands:**
  ```
  command 1
  command 2
  ```
- **Expected Result:** [What the user should see]

### Step 2: [Description of what this step accomplishes]
- **Commands:**
  ```
  command 1
  command 2
  ```
- **Expected Result:** [What the user should see]

[Additional steps as needed]

## Expected Output

Detailed description of the final expected outcome, including:
- Sample output text
- Expected visual elements
- Key indicators of success

## Troubleshooting Tips

- Common issue 1 and solution
- Common issue 2 and solution
- Resource for additional help

## Comprehension Check

1. What did you observe when [specific action]?
   - A) [Option A]
   - B) [Option B]
   - C) [Option C]
   - D) [Option D]
   
   **Correct Answer:** [Answer]
   **Explanation:** [Explanation]

## Summary

Brief summary of what was learned in the lab.
```

## Guidelines

1. **Clear Objectives**: Each lab should have specific, measurable learning objectives
2. **Detailed Steps**: Include all necessary commands and expected results
3. **Error Prevention**: Anticipate common errors and provide troubleshooting tips
4. **Verification**: Include ways to verify each step was completed correctly
5. **Accessibility**: Include alt-text for any visual elements
6. **Safety**: Include safety considerations for physical systems

## Best Practices

1. Test the lab completely before publishing
2. Use consistent formatting and terminology
3. Include expected time for completion
4. Provide clear prerequisites to avoid setup issues
5. Include variations for different platforms/hardware when applicable
